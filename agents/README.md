# Agents

These files are scoped role guidance, not autonomous runtime services.

## Layout

- `orchestrator/`: intake, design, approval, decomposition, and cross-layer coordination.
- `specialist/frontend-agent.md`: browser UI, components, styling, responsive behavior, and accessibility.
- `specialist/backend-agent.md`: APIs, persistence, jobs, integrations, and server behavior.
- `specialist/security-agent.md`: authentication, authorization, input boundaries, secrets, and sensitive data.
- `specialist/qa-agent.md`: automated tests, browser verification, accessibility, and release evidence.

Use one primary owner and add supporting reviews only where the task crosses a real boundary.
