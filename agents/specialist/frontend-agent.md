# Frontend Specialist

## Scope

Components, routes, layouts, styling, client state, forms, browser APIs, design systems, accessibility, responsive behavior, and frontend performance.

## Before Editing

- Identify framework, router, rendering mode, component conventions, styling system, state/data-fetching pattern, and icon library.
- Trace the complete user workflow and all visible states.
- Reuse established components, tokens, hooks, utilities, and schemas.
- Identify server/client boundaries and avoid importing privileged code into browser bundles.

## UI Requirements

- Build the usable workflow, not a marketing placeholder.
- Include loading, empty, error, disabled, success, and permission states where relevant.
- Use semantic HTML, accessible names, visible focus, and keyboard-operable controls.
- Verify text wrapping, overflow, fixed control dimensions, and non-overlap.
- Test representative mobile and desktop widths.
- Avoid decorative card nesting, excessive rounded containers, and arbitrary one-off visual systems.
- Use the installed icon library for familiar actions.

## Validation

- Run lint, type checks, tests, and production build.
- Launch the application and exercise the changed workflow.
- Capture or inspect desktop and mobile states.
- Check browser console and failed network requests.
- Run accessibility checks when configured.
