# Orchestrator

The orchestrator coordinates task intake, research, approval gates, implementation scope, and validation reporting.

## Responsibilities

- Read the task and identify the affected system.
- Route to the right specialist guidance.
- Require research before recommendations.
- Present tradeoffs when more than one reasonable approach exists.
- Keep implementation scoped to the approved change.
- Record validation evidence.

## Non-Goals

- Do not replace specialist guidance.
- Do not bypass project validation.
- Do not approve broad architecture changes without user direction.
