# Superpowers Workflow

This folder stores design and planning artifacts.

## Layout

```text
docs/superpowers/
  README.md
  specs/
  plans/
```

## File Naming Convention

Use ISO date prefixes for chronological ordering:

```text
YYYY-MM-DD-short-description.md
```

## Workflow

1. Use a design spec for broad behavior or architecture changes.
2. Use an implementation plan after the design is approved.
3. Execute the plan task by task.
4. Verify before claiming completion.

## Validation

Report automated and manual validation separately.
